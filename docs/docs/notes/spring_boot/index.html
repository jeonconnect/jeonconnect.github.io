spring boot

설정​

spring boot jar 배포시에 application.yml 배치​

현재 실행 디렉토리의 config 서브 디렉토리 또는 현재 실행 디렉토리 등에 application.yml 을 배치하면, jar 에 포함된 application.yml 의 설정을 override 한다.

  • [Spring Boot] 외부에서 설정 주입하기 - Increment (latera.kr)
  • 완전히 실행 가능한 Spring Boot 배포 (tistory.com)

gradle task 실행시에 application.yml 위치를 파라미터로 전달​

다음과 같이 한다. 이 경우에는 기본 application.yml 가 무시된다.

./gradlew bootRun --args="--spring.config.location=file:./some/path/some_custom_application.yml"

application.yml 에서 환경변수 사용​

${변수명} 형식을 사용해 환경변수가 가진 값으로 대치되도록 지정할 수 있다.

server:
  address: ${SERVER_IP}
  port: ${SERVER_PORT}

환경변수가 존재하지 않을 때의 기본사용값을 지정할 수도 있다.

logging:
  level:
    root: ${LOGGING_LEVEL_ROOT:info}

사용자의 홈 디렉토리는 ${user.home} 으로 참조할 수 있다.

  • java - Does application.yml support environment variables? - Stack Overflow

  • https://stackoverflow.com/a/61372261

nginx 연동​

특정 경로를 WAS 로 리다이렉트한다.

/etc/nginx/sites-available/xxxxxx.com 설정에 추가한다.

server {

        ...


        location /xxx {
                proxy_pass http://localhost:8080/xxx;
                proxy_ssl_server_name on;
        }

        ...

}

proxy_ssl_server_name on; 이 없으면 502 에러가 발생할 수 있다.

  • https://stackoverflow.com/a/49116797

pid 파일 자동 생성​

spring.pid.file 설정을 지정하면 해당 경로 파일에 실행된 프로세스의 pid 를 저장한다.

ApplicationPidFileWriter 를 어플리케이션 리스너로 등록해줘야 한다.

@SpringBootApplication
class SomeApplication

fun main(args: Array<String>) {
    SpringApplication(SomeApplication::class.java)
        .apply {
            addListeners(ApplicationPidFileWriter())
        }
        .run(*args)
}

pid 파일 생성 경로를 설정할 수 있다.

spring:
  pid:
    file: /xxx/yyy/zzz.pid

  • 스프링 부트 PID(Process Identifier) 파일 생성 방법 (jheloper.github.io)

종료 이벤트 처리​

  • SIGKILL 이 아니라 SIGTERM 시그널로 종료되어야 한다.

    kill -15 <pid>

  • ApplicationListener<ContextCloseEvent> 를 구현해서 등록한다.

    @Component
    class StringConnectShutdownListener : ApplicationListener<ContextClosedEvent> {

        override fun onApplicationEvent(event: ContextClosedEvent) {
            //TODO 종료시의 처리
        }
    }

JPA​

H2 Database 를 데이터소스로 사용​

build.gradle.kts 의존성 설정

implementation("org.springframework.boot:spring-boot-starter-data-jpa")
implementation("com.h2database:h2")

application.yml 설정

spring:
  h2:
    console:
      enabled: false
      path: "/h2"
  datasource:
    driver-class-name: org.h2.Driver
    url: jdbc:h2:mem:xxxx
    username : yyyy
    password : zzzz

메모리 DB 로 사용할 경우

spring.datasource.url=jdbc:h2:mem:xxxx

파일 DB 로 사용할 경우

spring.datasource.url=jdbc:h2:file:~/xxxx
  • Spring Boot With H2 Database | Baeldung
  • https://stackoverflow.com/a/47217202

kotlin 설정​

build.gradle.kts 에 JPA 사용을 위한 설정을 추가한다.

plugins {

    val kotlinVersion = "1.6.21"

    ...중략...
    
    kotlin("plugin.jpa") version "${kotlinVersion}"
    kotlin("plugin.allopen") version "${kotlinVersion}"
    kotlin("plugin.noarg") version "${kotlinVersion}"
}

noArg {
    annotation("javax.persistence.Entity")
}

allOpen {
    annotation("javax.persistence.Entity")
    annotation("javax.persistence.MappedSuperclass")
    annotation("javax.persistence.Embeddable")
}

정적 리소스 캐싱 금지 설정​

Spring MVC의 WebMvcConfigurer 인터페이스를 구현

@Configuration
public class WebConfig implements WebMvcConfigurer {

    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        registry.addResourceHandler("/some/**")
                .addResourceLocations("classpath:/static_some/")
                //.setCachePeriod(0)
                .setCacheControl(CacheControl.noStore())
                .resourceChain(true)
                .addResolver(new PathResourceResolver());
    }
}

  • 이 예제에서는 addResourceHandlers 메서드를 오버라이드하여 모든 요청(/some/**)에 대해 캐싱을 비활성화하는 핸들러를 등록한다.

  • .setCacheControl(CacheControl.noStore()) 설정이 적용되면 Spring MVC는 응답의 "Cache-Control" 헤더를 "no-store"로 설정한다.

  • 이 경우에 static_some 폴더는 resources 하위에 존재한다.

의존성 주입​

Bean 의 초기화 완료 이벤트 통지​

@PostConstruct 애너테이션은 빈 초기화 직후에 호출되는 메서드에 사용됩니다. Spring Boot에서 @Value로 지정한 멤버를 @PostConstruct를 사용하여 빈 초기화 후에 참조하는 예제는 다음과 같습니다.

import org.springframework.beans.factory.annotation.Value
import org.springframework.stereotype.Component
import javax.annotation.PostConstruct

@Component
class A {

    @Value("\${my.property}")
    private lateinit var myProperty: String

    @PostConstruct
    fun init() {
        println("myProperty: $myProperty")
    }
}

이 예제에서는 myProperty 필드에 @Value 애너테이션을 사용하여 속성 값을 주입하고, @PostConstruct 애너테이션을 사용하여 init() 메서드를 빈 초기화 직후에 호출되도록 설정했습니다. 이렇게 하면 init() 메서드에서 myProperty 값을 참조할 수 있습니다.

@Value 지정된 멤버를 생성자에서 참조​

@Value 애너테이션으로 지정한 멤버는 빈(Bean)이 생성되고 의존성 주입이 완료된 시점부터 참조할 수 있습니다. 생성자에서 @Value로 지정한 멤버에 접근하려면, 생성자 매개변수에 @Value 애너테이션을 붙여 사용해야 합니다.

예를 들어, A 클래스에 @Value 애너테이션으로 지정한 멤버를 생성자에서 참조하는 방법은 다음과 같습니다.

import org.springframework.beans.factory.annotation.Value
import org.springframework.stereotype.Component

@Component
class A(
    @Value("\${my.property}")
    private val myProperty: String
) {
    init {
        println("myProperty: $myProperty")
    }
}

이렇게 생성자 매개변수에 @Value 애너테이션을 사용하면, 클래스의 인스턴스가 생성되는 시점에 해당 속성의 값이 주입되어 사용할 수 있습니다. 단, 생성자 외부에서 선언된 필드에 @Value 애너테이션을 사용할 경우에는 생성자에서 해당 필드에 접근할 수 없습니다. 이 경우, @PostConstruct 애너테이션을 사용하여 빈 초기화 직후에 값을 참조하도록 설정할 수 있습니다.
