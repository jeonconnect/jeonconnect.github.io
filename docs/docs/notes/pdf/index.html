pdf

문서 구조​

PDF(Portable Document Format)는 Adobe Systems에서 개발한 파일 형식으로, 문서의 고정된 레이아웃과 글꼴, 이미지, 그래픽 요소 등을 포함하여 표현할 수 있습니다. PDF는 여러 플랫폼에서 독립적으로 사용할 수 있으며, 광범위한 용도로 사용되고 있습니다. PDF 문서의 기본 구조는 다음과 같습니다:

  1. 파일 헤더 (File Header): PDF 문서의 시작 부분으로, 파일의 버전 정보를 포함하고 있습니다. 일반적으로 '%PDF-1.x'의 형태를 가집니다 (여기서 x는 버전 번호).
  2. 바디 (Body): 이 부분은 PDF 문서의 내용을 구성하는 객체들을 포함하고 있습니다. 객체는 다양한 종류가 있으며, 페이지, 글꼴, 이미지, 양식 필드, 링크 등을 포함할 수 있습니다. 객체는 각각 고유한 번호와 세대 번호를 가지며, 그들 간의 관계를 나타내기도 합니다.
  3. 크로스-리퍼런스 테이블 (Cross-Reference Table): PDF 문서에서 객체들의 위치를 나타내는 테이블입니다. 이 테이블은 객체들의 바이트 오프셋을 포함하고 있어, 빠르게 객체에 접근할 수 있도록 도와줍니다.
  4. 트레일러 (Trailer): 문서의 마지막 부분으로, 크로스-리퍼런스 테이블의 위치, 루트 카탈로그 객체에 대한 참조 및 문서의 메타데이터를 포함하고 있습니다.
  5. EOF 마커 (EOF Marker): 파일의 끝을 나타내는 문자열로, '%EOF'와 같은 형태를 가집니다.

PDF 문서는 보통 다음과 같은 계층 구조를 가집니다:

  • 카탈로그 (Catalog): 문서의 루트 객체로서, 페이지 트리, 목차, 양식 등 다양한 문서 구조를 포함하고 있습니다.

  • 페이지 트리 (Page Tree): PDF 문서의 모든 페이지를 계층적으로 구성하는 트리 구조입니다. 각 노드는 여러 개의 페이지를 참조할 수 있습니다.

  • 페이지 객체 (Page Object): 개별 페이지의 내용과 속성을 포함하고 있습니다. 페이지의 레이아웃, 글꼴, 이미지, 텍스트 등의 정보를 포함한 콘텐츠 스트림을 참조합니다.

  • 콘텐츠 스트림 (Content Stream): 페이지의 콘텐츠를 그리기 위한 명령어와 데이터를 포함하는 스트림입니다. 콘텐츠 스트림은 텍스트, 이미지, 벡터 그래픽, 양식 객체 등의 그래픽 요소를 그리는 데 필요한 연산자와 피연산자를 포함합니다.

    • 리소스 딕셔너리 (Resource Dictionary): 페이지나 양식 객체에서 사용되는 공통 리소스를 포함하는 컨테이너입니다. 글꼴, 이미지, 색 공간, 패턴, 그래픽 상태 등의 리소스를 포함할 수 있습니다.
    • 인터랙티브 요소: PDF 문서는 인터랙티브 기능을 포함할 수 있으며, 링크, 주석, 양식 필드 등의 요소를 지원합니다.
      • 링크 (Link): 문서 내의 다른 위치나 외부 리소스로의 연결을 제공하는 인터랙티브 요소입니다.
      • 주석 (Annotation): 페이지에 추가 정보를 제공하는 오버레이 요소로, 텍스트 주석, 하이라이트, 스탬프, 사운드 등 다양한 형태가 있습니다.
      • 양식 필드 (Form Field): 사용자 입력을 수집하기 위한 인터랙티브 요소로, 텍스트 필드, 체크 박스, 라디오 버튼, 버튼, 드롭다운 목록 등이 있습니다.
    • 메타데이터 (Metadata): PDF 문서의 작성자, 제목, 주제, 키워드 등의 정보를 포함하는 데이터입니다. 메타데이터는 XML 기반의 XMP (Extensible Metadata Platform) 형식으로 저장됩니다.
    • 보안 기능: PDF는 문서의 저작권을 보호하고 무단 수정을 방지하기 위한 다양한 보안 기능을 제공합니다. 암호화, 디지털 서명, 접근 권한 설정 등의 기능을 지원합니다.

    이와 같은 구조와 요소들을 통해 PDF는 다양한 문서 형식에 대한 높은 호환성과 독립성을 제공하며, 사용자가 원하는 레이아웃과 디자인을 그대로 보존할 수 있습니다. 또한, 인쇄, 전자 서명, 양식 작성 등의 다양한 기능을 지원하여 전자 문서 작업에 편리함을 제공합니다.

shading 처리​

PDF의 Shading 처리는 그래픽 요소에 대한 색상 전환 및 그라데이션 효과를 생성하기 위한 기능입니다. Shading은 주로 벡터 기반의 그래픽에서 사용되며, 객체에 부드러운 색상 변화를 적용할 수 있습니다. PDF에서 지원하는 Shading 유형은 총 7가지로, 다음과 같은 구조를 가집니다:

  1. Function-based Shading (Type 1): 이 유형의 Shading은 두 개 이상의 입력 변수와 미리 정의된 수학 함수를 사용하여 색상 변화를 계산합니다. 사용자가 정의한 함수에 따라 다양한 그라데이션 효과를 생성할 수 있습니다.

  2. Axial Shading (Type 2): 이 유형은 시작점과 끝점 사이의 직선 경로를 따라 그라데이션 효과를 생성합니다. 시작점과 끝점의 색상은 미리 정의되어 있으며, 경로에 따라 색상이 부드럽게 변화합니다.

  3. Radial Shading (Type 3): 이 유형은 두 개의 원 사이에서 그라데이션 효과를 생성합니다. 두 원의 중심점과 반지름이 정의되어 있으며, 중심점부터 외부로 색상이 부드럽게 변화합니다.

  4. Free-form Gouraud-shaded Triangle Mesh (Type 4): 이 유형은 삼각형 메시를 사용하여 그라데이션 효과를 생성합니다. 각 삼각형의 꼭짓점에 색상이 할당되어 있으며, 삼각형 내부에서 색상이 Gouraud 쉐이딩 방식에 따라 부드럽게 보간됩니다.

  5. Lattice-form Gouraud-shaded Triangle Mesh (Type 5): 이 유형은 격자 형태의 삼각형 메시를 사용하여 그라데이션 효과를 생성합니다. 격자의 각 꼭짓점에 색상이 할당되어 있으며, 격자 내부에서 색상이 Gouraud 쉐이딩 방식에 따라 부드럽게 보간됩니다.

  6. Coons Patch Mesh (Type 6): 이 유형은 Coons Patch라 불리는 사각형 패치를 사용하여 그라데이션 효과를 생성합니다. 패치의 각 모서리에 색상이 할당되어 있으며, 패치 내부에서 색상이 보간됩니다. 이 유형은 복잡한 곡선 및 패턴에 적합한 그라데이션을 생성하는 데 사용됩니다.

  7. Tensor-product Patch Mesh (Type 7): 이 유형은 Tensor-product Patch라 불리는 사각형 패치를 사용하여 그라데이션 효과를 생성합니다. Coons Patch Mesh와 유사하지만, 보간 방식이 다릅니다. Tensor-product Patch Mesh는 패치의 각 모서리뿐만 아니라 패치의 가장자리에 위치한 제어점들에도 색상 정보를 할당합니다. 이를 통해 더욱 복잡한 곡선 및 패턴에 적합한 그라데이션을 생성할 수 있습니다.

    PDF에서 Shading 처리를 위한 객체는 Shading Dictionary라는 구조를 사용합니다. Shading Dictionary는 다음과 같은 정보를 포함합니다:

    • Shading Type: 위에서 설명한 7가지 Shading 유형 중 하나를 나타냅니다.
    • Color Space: 색상 공간을 정의하며, DeviceRGB, DeviceCMYK, DeviceGray 등의 색상 공간을 사용할 수 있습니다.
    • Background (optional): Shading이 적용되지 않은 영역의 배경색을 설정할 수 있습니다.
    • BBox (optional): Shading 영역의 경계 상자를 설정할 수 있습니다.
    • Anti-alias (optional): 안티에일리어싱 효과를 적용할지 여부를 설정할 수 있습니다.

    각 Shading 유형에 따라 추가적인 정보가 Shading Dictionary에 포함될 수 있습니다. 예를 들어, Axial Shading의 경우 시작점, 끝점, 시작점 색상, 끝점 색상 등의 정보가 필요합니다.

    Shading 처리를 사용하면 그래픽 요소에 다양한 그라데이션 효과를 적용할 수 있어, 그래픽의 품질과 복잡성을 높일 수 있습니다. 이러한 효과는 로고, 배경, 차트 등 다양한 디자인 요소에 활용됩니다.

예제​

PDF에서 Shading을 지정하는 예제를 제공하겠습니다. 여기서는 간단한 Axial Shading (Type 2)을 사용하여 직선 경로를 따라 그라데이션 효과를 생성하는 방법을 설명합니다.

먼저 Shading Dictionary를 생성합니다:

javascriptCopy code5 0 obj
<<
  /Type /Shading
  /ShadingType 2
  /ColorSpace /DeviceRGB
  /Coords [0 0 200 0]
  /Function 6 0 R
>>
endobj

이 예제에서는 Shading 객체를 생성하고, 5번 ID를 부여했습니다. Shading 유형으로는 Type 2 (Axial Shading)를 지정하였고, 색상 공간으로는 DeviceRGB를 사용했습니다. Coords 배열에서 [0 0 200 0]는 시작점 (0, 0)에서 끝점 (200, 0)까지의 직선 경로를 나타냅니다. Function 항목은 그라데이션 색상을 계산하는 데 사용되는 함수를 참조합니다. 여기서는 6번 객체에 정의된 함수를 사용하도록 설정했습니다.

다음으로, 그라데이션 색상을 계산하는 함수를 정의합니다:

cssCopy code6 0 obj
<<
  /FunctionType 2
  /Domain [0 1]
  /C0 [1 0 0]
  /C1 [0 0 1]
  /N 1
>>
endobj

이 예제에서는 6번 객체에 Function Type 2 (Exponential Interpolation)를 사용하여 색상을 보간하는 함수를 정의했습니다. Domain 배열에서 [0 1]은 함수의 입력 범위를 나타냅니다. C0에서는 시작점의 색상을 지정하였으며, 여기서는 빨간색 (RGB: 1, 0, 0)을 사용했습니다. C1에서는 끝점의 색상을 지정하였으며, 여기서는 파란색 (RGB: 0, 0, 1)을 사용했습니다. N 값은 지수를 나타냅니다. 이 경우에는 1을 사용하여 선형 보간이 수행됩니다.

마지막으로, Shading을 페이지에 적용합니다:

shellCopy code% ... 페이지 설정 생략 ...

/Contents 7 0 R

% ... 페이지 설정 생략 ...
cssCopy code7 0 obj
<< /Length 68 >>
stream
q
100 100 200 200 re
W
n
5 0 R sh
Q
endstream
endobj

여기서는 7번 객체에 페이지 콘텐츠 스트림을 정의하였습니다. re, W, n 명령어를 사용하여 100,100을 시작점으로 가로 200, 세로 200 크기의 직사각형 클리핑 영역을 생성합니다. 클리핑 영역을 설정한 후에는 5 0 R sh 명령어를 사용하여 앞서 정의한 Shading 객체를 적용합니다. 이렇게 하면 직사각형 영역에 그라데이션 효과가 적용됩니다. 마지막으로 Q 명령어를 사용하여 그래픽 상태를 복원합니다.

이 예제에서는 다음과 같은 순서로 PDF 요소가 배치됩니다:

  1. Shading Dictionary (5번 객체)를 생성하여 Shading 유형, 색상 공간, 경로 등을 설정합니다.
  2. 함수 객체 (6번 객체)를 생성하여 그라데이션 색상을 계산하는 데 사용되는 함수를 정의합니다.
  3. 콘텐츠 스트림 (7번 객체)를 생성하여 Shading을 페이지에 적용하고, 그라데이션 효과가 적용된 영역을 클리핑합니다.

이 예제를 통해 PDF에서 Shading을 지정하고 적용하는 방법에 대한 이해를 얻으셨길 바랍니다. 유사한 방법으로 다른 Shading 유형도 사용할 수 있으며, 다양한 그라데이션 효과를 생성할 수 있습니다.
