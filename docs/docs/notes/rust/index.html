dependency 로 git 저장소 지정​

~/.cargo/config 파일에 아래 내용을 추가한다.

[net]
git-fetch-with-cli = true

Cargo.toml 에 아래와 같이 의존성 지정한다.

xxx_lib = { git = "ssh://git@xxx.com/yyy/xxx_lib.git" }

아래 명령으로 해당 라이브러리만 다시 갱신할 수 있다.

cargo update -p xxx_lib

  • 모든 에러를 받을 수 있는 Error 와 Result 를 정의한다.
type AnyError = Box<dyn std::error::Error + Send + Sync + 'static>;
type AnyResult = Result<T, AnyError>

  • 반환 타입을 AnyResult<T> 로 지정하면 ? 연산자는 모든 오류 타입을 AnyResult 로 자동 변환한다.

            }

        }
    }

Iterator​

Rust의 Iterator trait은 다음과 같이 정의됩니다.

pub trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
    // 여기에 더 많은 메서드가 있지만, 질문에 초점을 맞추기 위해 생략하겠습니다.
}

여기서 type Item은 연관 타입(Associated Type)입니다. 연관 타입은 특정 trait과 관련된 타입을 정의할 때 사용되며, 이를 구현하는 구조체나 열거체에서 실제 타입을 지정합니다. Iterator trait에서 Item은 이터레이터가 반환하는 값의 타입을 나타냅니다.

trait Iterator<Item>의 경우, 이는 제네릭 타입(Generic Type)을 사용한 표현입니다. 제네릭 타입은 일반적으로 여러 타입에 대해 동작하는 코드를 작성할 때 사용되며, 특정 타입에 의존하지 않습니다.

두 표현의 주요 차이점은 다음과 같습니다.

  • 연관 타입(Associated Type)을 사용한 type Item: Iterator trait과 관련된 타입을 명시적으로 지정해야 합니다. 구현하는 구조체나 열거체에서 실제 타입을 지정할 수 있습니다.
  • 제네릭 타입(Generic Type)을 사용한 trait Iterator<Item>: Iterator trait를 구현할 때 타입 매개변수를 명시적으로 지정해야 합니다. 이렇게 하면 구현 시점에 사용할 수 있는 타입에 더 많은 유연성이 생깁니다.

연관 타입과 제네릭 타입 모두 코드를 추상화하는데 사용되지만, 어떤 상황에 적합한지에 따라 선택이 달라질 수 있습니다. 연관 타입은 주로 trait과 관련된 타입이 하나뿐일 때 사용되며, 제네릭 타입은 여러 타입에 동작하는 코드를 작성할 때 사용됩니다.

다음은 Rust에서 Iterator trait을 구현하는 간단한 예제입니다. 이 예제에서는 CountUp이라는 구조체를 정의하고, 그 구조체에 대해 Iterator trait을 구현합니다.

struct CountUp {
    current: i32,
    limit: i32,
}

impl Iterator for CountUp {
    type Item = i32; // 연관 타입을 지정합니다.

    fn next(&mut self) -> Option<Self::Item> {
        if self.current <= self.limit {
            let result = self.current;
            self.current += 1;
            Some(result)
        } else {
            None
        }
    }
}

fn main() {
    let count_up = CountUp {
        current: 1,
        limit: 5,
    };

    for number in count_up {
        println!("{}", number);
    }
}

이 코드는 1에서 5까지 숫자를 출력합니다. CountUp 구조체는 current와 limit 두 개의 필드를 가지며, 이터레이션 도중 현재 값과 상한값을 추적합니다. Iterator trait의 구현에서는 Item 타입을 i32로 지정하고, next() 메서드를 구현하여 숫자를 증가시키거나 이터레이션의 끝에 도달했을 때 None을 반환합니다.

Iterator 와 IntoIterator 의 차이​

Rust에서 Iterator와 IntoIterator는 관련되어 있지만 서로 다른 용도로 사용되는 두 개의 trait입니다.

  1. Iterator trait:

    Iterator trait은 객체를 반복할 수 있는 기능을 정의합니다. 이 특성을 구현하는 타입은 next() 메서드를 제공해야 하며, 이 메서드는 반복의 다음 항목을 반환하거나 더 이상 항목이 없을 경우 None을 반환합니다. Iterator trait은 또한 연관 타입 Item을 정의하며, 이는 반복되는 요소의 타입을 나타냅니다.

  2. IntoIterator trait:

    IntoIterator trait은 객체를 이터레이터로 변환하는 기능을 정의합니다. 이 특성을 구현하는 타입은 into_iter() 메서드를 제공해야 하며, 이 메서드는 해당 타입의 값을 해당 값의 이터레이터로 변환합니다. IntoIterator trait은 또한 연관 타입 IntoIter를 정의하며, 이는 반환되는 이터레이터의 타입을 나타냅니다.

간단히 요약하면, Iterator는 객체를 반복하는 데 필요한 메서드를 정의하는 반면, IntoIterator는 객체를 이터레이터로 변환하는 데 필요한 메서드를 정의합니다.

예를 들어, Rust의 표준 라이브러리에서 제공하는 Vec<T> 타입은 IntoIterator trait을 구현하고 있습니다. 따라서 Vec<T>를 이터레이터로 변환하여 요소를 반복할 수 있습니다.

fn main() {
    let vec = vec![1, 2, 3];

    for item in vec {
        println!("{}", item);
    }
}

위의 코드에서 for 루프는 vec이 구현하는 IntoIterator trait의 into_iter() 메서드를 내부적으로 호출하여 이터레이터를 얻고, 그 이터레이터의 Iterator trait의 next() 메서드를 사용하여 반복을 수행합니다.

Vec<T>는 Iterator trait을 직접 구현하지 않는데, 이는 몇 가지 이유 때문입니다.

  1. 소유권 및 불변성 보존: Iterator trait을 직접 구현하면 Vec<T>의 소유권을 다루거나 내부 데이터에 대한 불변성을 보존하기가 어려워집니다. Vec<T>는 컬렉션에 대한 소유권을 유지하면서 이터레이션을 수행할 수 있는 별도의 이터레이터를 제공하는 것이 더 좋은 설계입니다.
  2. 분리된 관심사: Vec<T>의 주요 목적은 메모리에 연속적인 배열을 저장하고 관리하는 것입니다. 이와 별도로, 반복 기능은 독립적인 관심사입니다. 이를 분리하여 관리하면 코드가 더 명확해지고, 이해하기 쉬워집니다.

이러한 이유로, Rust의 표준 라이브러리에서는 Vec<T>를 위한 이터레이터를 별도의 타입으로 제공합니다. 이는 std::slice::Iter와 std::slice::IterMut 타입으로, 각각 불변 참조와 가변 참조에 대한 이터레이터를 제공합니다. 이러한 이터레이터 타입은 Vec<T>에 대해 IntoIterator trait을 구현하여 사용할 수 있습니다.

예를 들어, 다음 코드는 Vec<T>에 대한 이터레이터를 얻고 사용하는 방법을 보여줍니다.

fn main() {
    let vec = vec![1, 2, 3];

    // 불변 참조에 대한 이터레이터를 얻습니다.
    let iter = vec.iter();

    for item in iter {
        println!("{}", item);
    }

    // 가변 참조에 대한 이터레이터를 얻습니다.
    let mut vec_mut = vec![4, 5, 6];
    let iter_mut = vec_mut.iter_mut();

    for item in iter_mut {
        *item += 1;
        println!("{}", item);
    }
}

이렇게 하면 Vec<T>의 역할과 이터레이터의 역할이 명확하게 구분되며, 소유권 및 불변성에 대한 관리가 쉬워집니다.

클래스 계층구조 표현​

Rust에서 클래스 계층 구조를 나타내려면 트레잇(trait), 구조체(struct) 및 열거형(enum)을 사용할 수 있습니다. GUI 위젯 트리의 예를 들어 설명하겠습니다.

먼저, 공통 행동을 정의하는 트레잇을 생성합니다.

trait Widget {
    fn draw(&self);
    // ... 다른 공통 메서드들
}

다음으로, 여러 위젯 유형을 나타내는 구조체를 생성합니다.

struct Button {
    // ... 버튼에 대한 속성 및 메서드
}

struct TextBox {
    // ... 텍스트 상자에 대한 속성 및 메서드
}

struct Label {
    // ... 레이블에 대한 속성 및 메서드
}

각 구조체에 대해 트레잇을 구현합니다.

impl Widget for Button {
    fn draw(&self) {
        // ... 버튼 그리기 코드
    }
}

impl Widget for TextBox {
    fn draw(&self) {
        // ... 텍스트 상자 그리기 코드
    }
}

impl Widget for Label {
    fn draw(&self) {
        // ... 레이블 그리기 코드
    }
}

이제 위젯 트리를 나타내는 열거형을 생성합니다.

enum WidgetNode {
    Button(Button),
    TextBox(TextBox),
    Label(Label),
}

WidgetNode 열거형은 다양한 위젯 유형 중 하나를 나타낼 수 있습니다. 이제 WidgetNode를 사용하여 위젯 트리를 구성할 수 있습니다.

struct Container {
    children: Vec<WidgetNode>,
}

impl Container {
    fn add_child(&mut self, child: WidgetNode) {
        self.children.push(child);
    }
}

Container 구조체는 여러 위젯을 포함할 수 있습니다. 이렇게 Rust에서 트레잇, 구조체, 열거형을 사용하여 클래스 계층 구조를 나타낼 수 있습니다. 이러한 방법을 사용하면 GUI 라이브러리에서 위젯 계층 구조를 표현할 수 있습니다.
