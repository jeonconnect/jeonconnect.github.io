export 와 export default​

export default와 export를 사용하는 것에 대한 선택은 대부분 프로젝트의 스타일 가이드나 개발자의 개인적인 선호에 따라 달라집니다. 그러나 export default class 대신 export class를 사용하라는 주장에 대해 몇 가지 이유를 들 수 있습니다.

  1. 명시적인 Import: export default를 사용하면 import할 때 어떤 이름을 사용하든 관계가 없습니다. 이는 코드를 읽는 사람에게 혼란을 줄 수 있습니다. 반면에 export class는 특정 이름으로 export하므로, 해당 이름을 사용하여 import해야 합니다. 이는 코드를 더 명확하게 만듭니다.

    // export default class
    export default class MyClass {}

    // 다른 파일에서 import 할 때
    import AnyNameYouWant from './MyClass';

    // export class
    export class MyClass {}

    // 다른 파일에서 import 할 때
    import { MyClass } from './MyClass';  // 반드시 MyClass를 사용해야 합니다.

  2. Autocomplete와 Refactoring: 대부분의 코드 에디터는 이름이 지정된 import를 더 잘 지원합니다. 즉, export class를 사용하면 코드 에디터의 자동 완성 기능이 더 잘 작동하고, 클래스 이름을 변경할 때에도 refactoring이 더 쉽습니다.

  3. Multiple Exports: export default는 파일당 하나만 가능합니다. 반면에 export는 여러 개를 사용할 수 있으므로, 한 파일에서 여러 클래스나 함수를 export할 필요가 있는 경우에 유용합니다.

이러한 이유로, export class를 사용하는 것이 더 좋을 수도 있습니다. 그러나 이것은 프로젝트나 개발 팀의 스타일 가이드에 따라 달라질 수 있으므로, 항상 해당 가이드를 따르는 것이 좋습니다.

설정 파일을 선택적으로 적용​

tsc​

TypeScript는 여러 tsconfig.json 파일을 사용하는 것을 지원합니다. 이를 통해 여러 설정 세트를 미리 준비하고 필요에 따라 선택적으로 사용할 수 있습니다.

이를 위해서는 각각의 설정에 해당하는 tsconfig.json 파일을 작성하고 이들을 프로젝트 폴더에 저장합니다. 그리고 tsc 커맨드를 실행할 때 -p 또는 --project 옵션을 사용하여 사용할 설정 파일을 지정합니다.

예를 들어, tsconfig.esnext.json과 tsconfig.commonjs.json 두 개의 설정 파일이 있다고 가정해봅시다. 각각 ESNext 모듈과 CommonJS 모듈을 사용하는 설정을 가지고 있다고 합시다. 이 경우, 해당 설정을 사용하여 프로젝트를 컴파일하려면 다음과 같은 커맨드를 사용할 수 있습니다:

ESNext 모듈 설정으로 컴파일하기:

tsc -p tsconfig.esnext.json

CommonJS 모듈 설정으로 컴파일하기:

tsc -p tsconfig.commonjs.json

이렇게 하면, 동일한 코드 베이스에 대해 서로 다른 설정을 적용하여 컴파일하는 것이 가능합니다. 이는 라이브러리를 작성하거나, 특정 환경에 맞추어 컴파일해야 하는 등의 상황에서 유용하게 사용할 수 있습니다.

ts-node​

ts-node는 TypeScript 코드를 직접 실행할 수 있게 해주는 도구로, tsc와 마찬가지로 -P 또는 --project 옵션을 사용하여 특정 설정 파일을 지정할 수 있습니다.

예를 들어, tsconfig.commonjs.json 설정 파일을 사용하려면 다음과 같이 ts-node를 실행할 수 있습니다:

bashCopy code
ts-node -P tsconfig.commonjs.json myfile.ts

위의 커맨드는 tsconfig.commonjs.json 설정을 사용하여 myfile.ts 파일을 실행합니다.

이처럼 ts-node를 사용하면 TypeScript 설정 파일을 지정하여 TypeScript 코드를 직접 실행할 수 있습니다. 이는 특히 개발 환경에서 유용하게 사용할 수 있습니다.
